ChatbotState

Represents a state that a Chatbot can be in. Has its own actions, answers and suggestions, separate from other states.

Functions for adding actions, answers and suggestions are provided, but functions for removing them aren't. This is because changing the behaviour of a state should not be done once it's set. If you need to change the behaviour of the Chatbot, create a new state with the appropriate actions, answers and suggestions and then change the Chatbot's current state to the new state.

Constructor

new ChatbotState(name);

Parameter Type Description
name string A string representing the name of the state. Must be unique (for this chatbot instance).

Properties

.name

String that was given to ChatbotState through constructor. This name will be used as a key in the map of all states in a Chatbot instance.
Type: string

.actions

A map containing actions assigned to intents. The key is the intent, the value is the action. When the handle method is called with an intent from this map, the corresponding action is called with the first parameter being this ChatbotState.
Type: Map<string, function>

.answers

A map containing answers assigned to intents. The key is the intent, the value is an array of answers (multiple answers can be added for the same intent). When the handle method is called with an intent from this map, all corresponding answers (the entire array) are returned as part of the result. Note: if using the import/export functionality, answers should be strings (because they are imported and exported as such).
Type: Map<string, Array<any>>

.suggestions

A map containing suggestions assigned to intents. The key is the intent, the value is an array of suggestions (multiple suggestions can be added for the same intent). When the handle method is called with an intent from this map, all corresponding suggestions (the entire array) are returned as part of the result. Note: if using the import/export functionality, suggestions should be strings (because they are imported and exported as such).
Type: Map<string, Array<any>>

Methods

.addAction( intent, action )

Used for adding an action that will be executed when the given intent is recognized. Only one action is allowed per intent, so calling this function again with the same intent will result in overwriting the old action with the new one. Returns true if there was no action assigned to the intent previously, otherwise returns false.
Parameter Type Description
intent string Intent to wait for
action function The action that will be executed

Returns: boolean

.addAnswer( intent, answer )

Used for adding answers to return for the given intent. If the provided answer is of the type Array, all elements of the Array are added, instead of adding the Array itself as an element. Returns true if there were no answers for given intent previously, otherwise returns false.
Parameter Type Description
intent string Intent to wait for
answer any or Array<any> The answer (or answers) to return for given intent

Returns: boolean

.addSuggestion( intent, suggestion )

Used for adding suggestions to return for the given intent. If the provided suggestion is of the type Array, all elements of the Array are added, instead of adding the Array itself as an element. Returns true if there were no suggestions for given intent previously, otherwise returns false.
Parameter Type Description
intent string Intent to wait for
suggestion any or Array<any> The suggestion (or suggestions) to return for given intent

Returns: boolean

.handle( intent )

Handles the intent. Called by the Chatbot when this state is the current state. Executes actions for the intent (if any exist), and returns the intent along with the corresponding answers and suggestions (if any are defined for it).
Parameter Type Description
intent string Intent to handle

Returns: Promise<{intent, answers, suggestions}>
